# AQA Approval — Requirements Checklist

> Working document. Check items off as we complete them. Items marked `[TBC]` need AQA to confirm exact requirements.

## How to read this checklist

Each item is one piece of documentation or evidence we need to be able to produce on request. AQA's public guidance does not give an itemised "bring these 37 documents" list, so this is a conservative union of (a) what AQA's published policies imply, (b) what peer publishers report being asked for, and (c) what any sane exam board would want to see from a new partner.

Priority legend:
- **P0** — must have before first serious conversation with AQA
- **P1** — must have before formal proposal submission
- **P2** — must have before Stage 2 content review
- **P3** — must have before Stage 4 sign-off

---

- [ ] **A4.** Trading history / years active
- [ ] **A5.** Bank reference or equivalent financial standing evidence `[TBC with AQA]`
- [ ] **A6.** Insurance certificates:
  - [ ] Professional indemnity (minimum £[PLACEHOLDER] cover)
  - [ ] Public liability
  - [ ] Cyber / data breach
- [ ] **A7.** Data protection registration (ICO)
- [ ] **A8.** GDPR / UK GDPR compliance statement

## B. Team and editorial credibility (P0)

- [ ] **B1.** Editorial team CVs, emphasising subject specialism
- [ ] **B2.** List of contributors with current or past AQA examining experience — see `03-evidence/examiner-credentials.md`
- [ ] **B3.** Conflict-of-interest declarations for any AQA examiner on the team (AQA's author CoI policy requires this)
- [ ] **B4.** Named editorial lead accountable to AQA during the process
- [ ] **B5.** Subject matter expert bench for English Literature
- [ ] **B6.** Subject matter expert bench for English Language
- [ ] **B7.** Copy-editing and proofreading process document

## C. Product description (P1)

- [ ] **C1.** Platform description: what The English Hub is, how it works, who uses it
- [ ] **C2.** Technical description: web platform, device support, offline access
- [ ] **C3.** User journey documentation for a GCSE student
- [ ] **C4.** User journey documentation for a teacher
- [ ] **C5.** Content inventory — every page, quiz, video, worked example
- [ ] **C6.** Content taxonomy — how content is organised and navigated
- [ ] **C7.** Eligibility positioning document: argument for why a digital-native platform qualifies under AQA's "print and digital textbooks" policy — or proposal for a print/PDF companion artefact that clearly qualifies

## D. Specification alignment (P1)

- [ ] **D1.** Full mapping of English Hub content to AQA GCSE English Literature (8702) specification — section by section
- [ ] **D2.** Full mapping to AQA GCSE English Language (8700) specification
- [ ] **D3.** Assessment Objective mapping: which content addresses which AO, with evidence
- [ ] **D4.** Paper-by-paper coverage: Literature Paper 1, Literature Paper 2, Language Paper 1, Language Paper 2, Spoken Language Endorsement
- [ ] **D5.** Set text coverage matrix (Shakespeare / 19th century novel / modern texts / poetry anthology — all options offered)
- [ ] **D6.** Gaps analysis — honest list of where we do not yet cover specification content
- [ ] **D7.** Remediation plan for gaps (what we'll write to close them)
- [ ] **D8.** Command-word glossary alignment — our tasks use the same command language as AQA papers
- [ ] **D9.** Mark scheme alignment evidence — our model answers match the shape of AQA mark schemes

## E. Pedagogy (P2)

- [ ] **E1.** Pedagogy statement: what we believe about how students learn English at GCSE
- [ ] **E2.** Scaffolding and progression model
- [ ] **E3.** Retrieval practice and spacing approach (if applicable)
- [ ] **E4.** Accessibility and SEND considerations — reading level, alt text, dyslexia-friendly patterns
- [ ] **E5.** Differentiation evidence — higher / foundation, stretch content
- [ ] **E6.** Avoiding "teach to the test" reductionism — statement and evidence

## F. Accuracy and quality assurance (P2)

- [ ] **F1.** Editorial workflow diagram: writer → reviewer → subject lead → sign-off
- [ ] **F2.** Fact-check process
- [ ] **F3.** Source verification policy — what counts as a citable source for literary criticism, linguistics, historical context
- [ ] **F4.** Error correction policy — what happens when an error is spotted in live content
- [ ] **F5.** Specification update process — what happens when AQA updates the specification
- [ ] **F6.** Change log / audit trail for content updates
- [ ] **F7.** Independent subject review process (second pair of eyes)

## G. Safeguarding (P2)

- [ ] **G3.** Age verification and age-appropriate content controls
- [ ] **G4.** Process for handling sensitive content in set texts (e.g. violence, self-harm, racial language in historical texts)
- [ ] **G5.** Designated Safeguarding Lead (DSL) or equivalent named
- [ ] **G6.** DBS check status for editorial staff who review content for minors `[TBC]`

## K. Sample content (P2)

AQA will want to see actual content, not just descriptions. We should pre-package:

- [ ] **K1.** One full set-text study unit (e.g. a Shakespeare play we cover)
- [ ] **K2.** One full poem study (from the AQA anthology)
- [ ] **K3.** One full Language Paper 1 unseen-extract teaching sequence
- [ ] **K4.** One full Language Paper 2 comparison teaching sequence
- [ ] **K5.** One full Spoken Language Endorsement guidance resource
- [ ] **K6.** One worked model answer per paper at Grade 4, Grade 6, Grade 8 level
- [ ] **K7.** A sample quiz with mark scheme
- [ ] **K8.** A sample teacher-facing resource (lesson plan or scheme of work)

## How to use this in practice

1. Assign each section to an owner.
2. Target P0 in month 1, P1 in months 2–3, P2 ready before Stage 2 review, P3 ready before Stage 4 sign-off.
3. Do not wait for AQA to ask — peer publishers report that the publishers who move fastest are the ones who anticipate.
4. Track progress in `06-tracking/application-tracker.md`.
