### Webhook secret

Generated by `openssl rand -base64 48` and saved (one line, no trailing
newline) to `secrets/revenuecat-webhook-secret.txt`. The token is the
part **after** "Bearer " in the Authorization header.

**Founder must set this in Vercel production env:**

```
REVENUECAT_WEBHOOK_SECRET=<contents of secrets/revenuecat-webhook-secret.txt>
```

Until that env var is set in production, `verifyWebhookSignature()` in
`src/lib/revenuecat/verify.ts` returns false for every delivery and the
webhook handler responds 401. RC will retry with exponential backoff
for 24 hours, so as long as the env var lands within a day of the
first event, no events are lost.

The webhook entry was created on 2026-05-01. If the dashboard shows
"failed" or "unauthorised" deliveries before the env var is set,
that's expected — they will succeed on retry once the env var is in.

## Outstanding RC follow-ups (post-launch)

- **Android key**: when ready to ship to Google Play, add a Google
  Play app configuration in RC. The Android public SDK key (starts
  `goog_…`) goes into `mobile/app.json` `extra.revenuecatAndroidApiKey`.
- **Sandbox testing**: create RC sandbox testers (Project Settings →
  Sandbox) for QA before TestFlight invites real users. RC sandbox
  testers are different from Apple sandbox testers — both are needed
  for end-to-end purchase replay.
- **Paywall builder**: optional — RC's no-code paywall builder can
  replace `mobile/app/paywall.tsx` if we want remote-config'd paywall
  experiments. Not needed for v1.
- **Promotional grants**: when reviewer feedback or support cases
  arise, use Customer → Promotional Entitlement to grant `premium`
  for free, time-bounded. Used for the Apple App Review demo account
  to avoid the "Upgrade" CTA showing on the native account screen
  (see `docs/APPLE_REVIEWER_ACCOUNT.md`).
